# AGENTS.md

## Purpose

This repository contains the Stumpwizards Ham Study Guide browser app and related assets. Future agents should treat the browser app as the primary maintained product unless the user explicitly asks for Windows app work.

## Source Of Truth

- Browser app source HTML: `site/index.html`
- Browser app source CSS: `site/src/ham-study.css`
- Browser app source JS: `site/src/ham-study-app.js`
- Browser app source data: `site/content/question-bank.json`
- Browser app source image asset: `site/Stumpwizard.png`
- Static site build script: `tools/build-site.mjs`

## Generated Output

- `public/` is generated output only.
- Do not hand-edit files under `public/`.
- Rebuild `public/` with `npm run build` after changing anything under `site/` or `tools/build-site.mjs`.
- `public/` is uploaded by GitHub Actions for GitHub Pages deployment.

## GitHub Pages Contract

- Pages is served from the GitHub Actions artifact, not from the repository branch root.
- Do not reintroduce root-level `index.html`, `404.html`, or branch-based Pages fallbacks unless the user explicitly asks for that.
- The deploy workflow builds the app and uploads `public/`.
- `public/.nojekyll` is generated by the build script and should not be tracked separately.

## App Behavior To Preserve

- Keep all 112 original questions.
- The study guide and interactive Q&A app are the deployable product.
- The Ohm’s Law calculator belongs directly below the Ohm’s Law graphic.
- `public/404.html` is intentionally a custom error page, not a copy of the main app.
  - It should show the Stumpwizard image.
  - It should say: `Something mysterious happened`.
  - It should link back to the app index page.

## Working Rules

- Prefer editing files under `site/` and `tools/`.
- If you change app structure or assets, verify `tools/build-site.mjs` still emits a complete `public/` tree.
- If you change question content or exam metadata, update only `site/content/question-bank.json`.
- Do not move app content into `README.md`.
- Do not add duplicate JS variants or alternate HTML entrypoints unless the user explicitly requests them.

## Local Commands

- Install dependencies: `npm ci`
- Build the Pages app: `npm run build`

## Verification

After relevant changes, run:

```bash
npm run build
```

Then verify at minimum:

- `public/index.html` exists
- `public/404.html` exists
- `public/src/ham-study-app.js` exists
- `public/src/ham-study.css` exists
- `public/content/question-bank.json` exists
- `public/Stumpwizard.png` exists

## Notes

- `README.md` is repository documentation only.
- If the user asks about the live Pages site, remember that GitHub Pages behavior can lag behind a local change until the Actions deploy finishes.
